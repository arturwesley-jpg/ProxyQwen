COMO USAR
--------------------------------------------------------------------------------

1. INICIAR SERVIDOR
   npm start

2. VERIFICAR HEALTH
   curl http://localhost:3000/health | jq .

3. VERIFICAR SESSÕES
   curl http://localhost:3000/sessions/stats | jq .

4. ENVIAR REQUEST COM CONVERSATION_ID (RECOMENDADO)
   curl http://localhost:3000/v1/chat/completions \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer ***     -d '{
       "model": "qwen3.7-plus",
       "conversation_id": "minha-conversa-123",
       "messages": [{"role": "user", "content": "Olá!"}],
       "stream": true
     }'

5. MONITORAR AGENTE
   ./monitor-agent.sh

--------------------------------------------------------------------------------
CONFIGURAÇÃO NO HERMES
--------------------------------------------------------------------------------

No config.yaml do Hermes:

```yaml
custom_providers:
  qwenproxy:
    base_url: "http://localhost:3000/v1"
    api_key: "***
    models:
      - qwen3.7-plus
```

Para aproveitar a session persistence, o Hermes deve enviar "conversation_id"
em cada request. Se não enviar, o proxy gera um automaticamente.

--------------------------------------------------------------------------------
TROUBLESHOOTING
--------------------------------------------------------------------------------

❌ Sessões não sendo reutilizadas
   → Verificar se conversation_id está sendo enviado
   → Ver logs: [Chat] Reusing session
   → Ver estatísticas: curl http://localhost:3000/sessions/stats

❌ Performance lenta
   → Verificar warm pool: grep "Warmed up" qwenproxy.log
   → Aumentar WARM_POOL_SIZE em src/services/qwen.ts
   → Verificar compressão: curl -H "Accept-Encoding: gzip" -I http://localhost:3000/health

❌ Contexto sendo truncado cedo
   → Aumentar preserveRecentMessages em src/routes/chat.ts
   → Verificar model context window em src/core/model-registry.ts
